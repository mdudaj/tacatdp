# User Stories: Multi-Project Monitoring Platform

## MP-US-01: Multiple projects

As a platform admin, I want each monitoring project to be a separate project record so that TACATDP and future programs can share the same platform without mixing definitions or data.

## MP-US-02: Versioned instruments

As a project manager, I want instruments/forms to have versions so that I can update a survey without corrupting existing submissions.

## MP-US-03: Repeating groups

As a form designer, I want groups to repeat so that I can collect multiple crops, household members, adverse events, visits, or cost lines without adding fixed columns for every possible instance.

## MP-US-04: Multi-select answers

As an analyst, I want multi-select answers normalized as rows so that I can count and filter each selected choice reliably.

## MP-US-05: Longitudinal entities

As a monitoring officer, I want submissions to link to a farmer, site, organization, or facility over time so that baseline and follow-up data are connected.

## MP-US-06: Controlled variables

As a data manager, I want controlled vocabularies with stable codes and labels so that choices are reusable, governed, and exportable across projects.

## MP-US-07: Project-specific exports

As an analyst, I want project-specific wide exports so that normalized storage does not prevent ordinary reporting and Power BI use.
