# Definition of Done: Canvas Metadata Renderer MVP

The Canvas renderer MVP is done when:

- A user can open the app without custom login.
- The seeded assigned form is visible to the assigned user.
- The seeded form renders from Dataverse metadata, not hard-coded screen fields.
- The user can enter answers for all seeded MVP field types.
- Required validation blocks incomplete submit.
- Save Draft persists `Submissions` and `SubmissionAnswers`.
- Submit changes the submission status to `Submitted`.
- History shows the user's own draft/submitted rows.
- Draft and Submitted rows are editable; Locked rows are read-only.
- One file/photo attachment can be saved through `SubmissionFiles`.
- GPS is either working or explicitly deferred with no impact to the core slice.
- The app is included in the dev solution.
- Known delegation warnings and unresolved risks are documented.
